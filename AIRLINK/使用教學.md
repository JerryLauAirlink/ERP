# AIRLINK ERP 使用教學

繁體中文（台灣）· 建議操作順序與常見錯誤說明

## 標準業務流程

```
客戶 → 報價單（Accepted）→ 工作 → 客戶 PO → 應付（付供應商）→ 工作完成 → 應收（向客戶開立發票）
```

---

## 零、開始之前（管理員／Root）

1. 登入後，確認右上角 **地區** 是否正確（HK／TW／SG 等）。
2. **設定 → 一般**：介面語言、時區。
3. **設定 → 備份與還原**：輸入雲端同步金鑰，啟用即時同步。
4. **設定 → 用戶管理**：建立帳號與模組權限、地區權限。

---

## 一、主檔資料（第一步）

### 1. 客戶（最優先）

- 路徑：**客戶 → + 新增客戶**
- 系統自動建議下一個客戶編號（如 SC057），可手動修改。
- **BU**：同一公司、不同聯絡人或工作時勾選，並填 BU 編號。

### 2. 供應商（可與客戶並行）

- 路徑：**供應商 → + 新增供應商**
- 應付單付款對象；建議填寫銀行、SWIFT、財務聯絡人及電子郵件。

> **注意**：一定要先有客戶，才能建立報價單與工作。

---

## 二、業務流程（嚴格順序）

| 步驟 | 模組 | 說明 |
|------|------|------|
| ① | 報價單 | 選客戶；連結工作前狀態須為 **Accepted** |
| ② | 工作 | 選客戶、連結報價單、填客戶 PO 明細 |
| ③ | 應付 AP | 連結工作 → 選供應商 → 記錄發票（可分期） |
| ④ | 工作狀態 | 完成後改為 **Completed** |
| ⑤ | 應收 AR | 連結工作 → 向客戶開立發票 |

---

## 三、報表

- **月報表**：修改工作／應收後須點 **同步月報**。
- **儀表板／總覽**：自動彙總應收應付。
- **進行中清單**：追蹤 PO 餘額與開票進度。

---

## 四、Excel 匯入順序

1. 客戶 → 2. 供應商 → 3. 報價單 → 4. 工作 → 5. 應收 → 6. 應付

匯入前確認右上角地區正確。

---

## 五、常見錯誤（請避免）

- 未建客戶就建工作／報價
- 報價未 Accepted 就連結工作
- 未建供應商就建應付（無法自動帶銀行資料）
- 應付未連結工作
- 改資料後未同步月報
- 地區選錯導致看不到資料
- 未啟用雲端同步，只靠本機瀏覽器

---

## 六、快速檢查清單

- [ ] 客戶已建立、編號唯一
- [ ] 報價 Accepted（若要連工作）
- [ ] 工作已選客戶、PO 完整
- [ ] 應付已連結工作與供應商
- [ ] 工作完成後已開應收
- [ ] 月報已同步
- [ ] 雲端同步正常
