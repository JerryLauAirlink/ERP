# AIRLINK ERP 使用教學

繁體中文（台灣）· 建議操作順序與常見錯誤說明  
對應版本：`airlink-2026-07-14e` 起

## 標準業務流程

```
客戶 → 報價單（Accepted）→ 工作 → 客戶 PO → 供應商採購單 Vendor PO → 應付（付供應商）→ 工作完成 → 應收（向客戶開立發票）
```

側欄順序參考：客戶 → 報價單 → 工作 → **供應商採購單** → 應收 → 應付 → 供應商

---

## 零、開始之前（管理員／Root）

1. 登入後，確認右上角 **地區** 是否正確（HK／TW／SG／MY／ID）。**新增資料會標記在目前地區**（選 ALL 時會落到你有權限的第一個地區）。
2. **設定 → 一般**：介面語言（中／英）、時區、公司名稱。
3. **設定 → 備份與還原**：
   - 輸入雲端同步金鑰（與 Vercel `ERP_SYNC_SECRET` 相同；若伺服器已設定，多數同事會自動帶入，不必人手入）。
   - 啟用即時同步後，重要操作（刪用戶、大批匯入）可按 **立即同步**。
4. **設定 → 用戶管理**：建立帳號、模組權限、地區權限。刪除用戶後請確認出現「已從雲端移除」；新瀏覽器不應再見到已刪帳號。

## 一、主檔資料（第一步）

### 1. 客戶（最優先）

- 路徑：**客戶 → + 新增客戶**
- 系統自動建議下一個客戶編號（如 SC057），可手動修改。
- **Invoice Title**：開立應收發票時可顯示的抬頭。
- **BU**：同一公司、不同聯絡人或工作時勾選，並填 BU 編號。
- **付款條件**：可快速選 30／45／60／90 Days，或自訂文字。

### 2. 供應商（可與客戶並行）

- 路徑：**供應商 → + 新增供應商**
- 欄位順序對齊 Excel：供應商編號、公司名稱、GST、負責人、地址、郵遞區號、公司電話、主要聯絡人、電郵、手機、職稱、銀行、分行、帳號、SWIFT、信用條件。
- 應付單付款對象；建議填齊銀行／SWIFT，開立 AP 時可自動帶入。

### 3. 搬去其他地區（加錯地區時）

若同事加錯地區（例如台灣公司出現在 SG）：

1. 開啟該筆 **客戶／工作／報價單／供應商／供應商採購單**
2. 按 **修改（Edit）**
3. 表單最上方 **「搬去其他地區」** 改成正確地區（如 TW）
4. 儲存（系統會確認一次）
5. 右上角切到正確地區即可看到

> **注意**：搬 Client **不會**自動搬相關 Job／Invoice／PO；那些記錄要各自 Edit 搬。

## 二、業務流程（嚴格順序）

| 步驟 | 模組 | 說明 |
|------|------|------|
| ① | 報價單 | 選客戶；狀態改為 **Accepted** 後可連結工作。Accepted 時可自動開啟「新增工作」。 |
| ② | 工作 | 選客戶、連結報價單（或勾「無報價單」自動編號如 HK00001）、填客戶 PO 明細。詳情可見 Client BU、Vendor、Vendor PO。 |
| ③ | 供應商採購單 Vendor PO | 見下方「Vendor PO」欄位說明 |
| ④ | 應付 AP | 連結工作 → 選供應商 → 可填 **AIRLINK PO** 與 **PO Amount** → 記錄供應商發票 |
| ⑤ | 工作狀態 | 完成後改為 **Completed** |
| ⑥ | 應收 AR | 連結工作 → 向客戶開立發票 |

點擊詳情內的藍色連結（報價／工作／發票／Vendor PO／供應商）可直接開啟對應詳情，不必只靠列表搜尋。

---

## 三、供應商採購單（Vendor PO）

路徑：**供應商採購單 → + 新增供應商採購單**

欄位順序（與 Excel 一致）：

| 欄位 | 說明 |
|------|------|
| Vendor Code | 供應商編號（選供應商後自動帶入） |
| Name | 供應商名稱 |
| For Project/ Inventory | Project 或 Inventory |
| Type | Material／Labour／Material and Labour／Service／Equipment／Subcontract／Other |
| Requested Delivery Date | 要求交貨日期 |
| Balance | PO 金額 − 已連結 AP「已收貨金額」合計（partial shipment 剩餘） |
| Airlink PO # | 唯一採購單號 |
| PO Date | 採購單日期 |
| Airlink PO Currency | 採購單幣別 |
| Airlink PO Amt | 原幣金額 |
| Airlink PO Amt in {當地幣} | 依**目前地區本位幣**換算（如 MY→MYR、SG→SGD）；可手動覆寫 |

可選填連結 **Job No**；連結後在「工作詳情」會顯示該 Vendor PO。

---

## 四、報表

- **月報表**：修改工作／應收後須點 **同步月報**。
- **儀表板／總覽**：自動彙總應收應付。
- **進行中清單**：追蹤 PO 餘額與開票進度。
- **管理層報表**：依已完成工作與 Accepted 報價分析毛利。

## 五、Excel 匯入順序

1. 客戶 → 2. 供應商 → 3. 報價單 → 4. 工作 → 5. **供應商採購單** → 6. 應收 → 7. 應付

匯入前確認右上角地區正確。建議用各模組的 **Download Template** 對齊欄位標題。

**工作（Job）匯入提示：** 填 **Customer No** 即可，不必填 Client Company（系統會依客戶主檔自動對應公司名稱）。匯出時也會帶出 Customer No。若公司名稱後來要改，到客戶主檔修改即可。

**無正式報價：** 新增／修改工作 → 已連結報價單區塊勾「無報價單」→ 系統依地區自動給號（HK00001、TW00001、SG00001…）並在儲存時建立 Accepted 佔位報價單。

## 六、常見錯誤（請避免）

- 未建客戶就建工作／報價
- 報價未 Accepted 就連結工作
- 未建供應商就建應付／Vendor PO
- 應付未連結工作
- **地區選錯**（用「搬去其他地區」修正，不要重複建一筆）
- 改資料後未同步月報
- 刪用戶後未等雲端成功／未按「立即同步」→ 新瀏覽器可能還原舊用戶
- 未啟用雲端同步，只靠本機瀏覽器

---

## 七、快速檢查清單

- [ ] 右上角地區正確；加錯可用「搬去其他地區」
- [ ] 客戶已建立、編號唯一；BU／Invoice Title 如需要已填
- [ ] 報價 Accepted（若要連工作）
- [ ] 工作已選客戶、PO 完整
- [ ] Vendor PO 欄位齊全；當地幣金額正確
- [ ] 應付已連結工作、供應商，並填 AIRLINK PO（如適用）
- [ ] 工作完成後已開應收
- [ ] 月報已同步；雲端同步／立即同步正常
