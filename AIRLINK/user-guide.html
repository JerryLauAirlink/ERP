AIRLINK ERP 使用教學

繁體中文（台灣）· 建議操作順序與常見錯誤說明

標準業務流程（請依序進行）
客戶 → 報價單（Accepted）→ 工作 → 客戶 PO → 應付（付供應商）→ 工作完成 → 應收（向客戶開立發票）

零、開始之前（管理員／Root）

  1. 登入後，確認右上角 地區 是否正確（HK／TW／SG 等）。新增資料會標記在目前地區。
  2. 前往 設定 → 一般：選擇介面語言、時區。
  3. 前往 設定 → 備份與還原：輸入雲端同步金鑰，啟用即時同步（與 Vercel 環境變數 ERP_SYNC_SECRET 相同）。
  4. Root 管理員在 設定 → 用戶管理 建立帳號，並設定各模組的檢視／新增／修改／刪除／匯入／匯出權限及地區權限。

一、主檔資料（第一步）

主檔必須先建立，後續報價、工作、應收應付才能正確連結。

1. 客戶（最優先）

  • 路徑：客戶 → + 新增客戶
  • 系統會自動建議下一個客戶編號（例如 SC056 之後為 SC057），仍可手動修改。
  • BU 勾選：同一公司名稱、不同聯絡人或不同工作時，可勾選 BU 並填寫 BU 編號；每筆仍須有獨立的客戶編號。

2. 供應商（可與客戶並行建立）

  • 路徑：供應商 → + 新增供應商
  • 應付單（AP）的付款對象。建議先建立常用供應商，並填寫銀行、SWIFT、財務聯絡人及電子郵件。
可並行：客戶與供應商可同時建立，但一定要先有任何一筆客戶，才能建立報價單與工作。

二、業務流程（嚴格順序）

步驟模組說明
①報價單選擇客戶、填寫金額與幣別。要連結至工作時，狀態須為 Accepted。
②工作選擇客戶，可連結一張或多張 Accepted 報價單。填寫客戶 PO 明細（PO 編號、金額、收單日期）。
③應付 AP連結工作 → 選擇供應商 → 記錄供應商發票。可分期（Deposit／Progress／Balance 等）。輸入付款日期即標記已付款。
④工作狀態完成交付後，將工作狀態改為 Completed。
⑤應收 AR連結工作 → 向客戶開立發票。建議在工作完成後開立；系統亦允許進行中開立分期發票。

三、報表與追蹤

  • 進行中清單：追蹤未結案工作與 PO 餘額。
  • 儀表板／總覽：自動從應收、應付彙總現金狀況。
  • 月報表：修改工作或應收後，須點擊 同步月報（或於 PO 收單／AR 發票分頁分別同步），報表數字才會更新。
  • 管理層報表：依已完成工作與 Accepted 報價分析毛利（需有對應應收／應付資料）。

四、Excel 匯入順序

若使用匯入功能，請依下列順序匯入，否則會出現「找不到客戶／工作」等錯誤：

  1. 客戶
  2. 供應商
  3. 報價單
  4. 工作
  5. 應收 AR
  6. 應付 AP

匯入前請確認右上角地區正確；匯入後若已啟用雲端同步，請等待狀態列顯示同步成功。

五、常見錯誤順序（請避免）

  • 未建客戶就建工作或報價 → 無法選擇客戶，匯入亦會失敗。
  • 報價單狀態非 Accepted 就連結工作 → 系統會拒絕連結。
  • 未建供應商就建應付 → 可手動輸入公司名稱，但無法自動帶入銀行與 SWIFT。
  • 應付未連結工作 → 工作詳情與月報無法正確歸集成本。
  • 修改資料後未同步月報 → 月報表數字停留在舊資料。
  • 地區選錯 → 資料存在其他地區，切換至 ALL 或正確地區才看得到。
  • 兩台電腦使用不同瀏覽器設定 → 請確認已啟用雲端同步，不要只依賴本機瀏覽器快取。

六、權限與多人使用

  • 僅有新增／修改／刪除／匯入權限的帳號，變更才會上傳至雲端。
  • 唯讀帳號只能下載資料，不會覆寫他人修改。
  • 同一帳號可在多台電腦登入；請以雲端資料為準，並在兩台電腦都執行強制重新整理（Ctrl+Shift+R）。

七、快速檢查清單

  1. 客戶已建立？客戶編號是否唯一？
  2. 報價單是否 Accepted（若要連結工作）？
  3. 工作是否已選客戶、PO 明細是否完整？
  4. 應付是否連結工作與供應商？
  5. 工作完成後是否已開應收發票？
  6. 月報是否已按「同步月報」？
  7. 雲端同步狀態是否正常？
